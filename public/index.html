Descrizione

In YANSA ogni appunto ha il proprio luogo e il proprio spazio: che sia nel feed degli account che si seguono, che sia solo nel proprio perché si vuole tenere privato, YANSA troverà lo spazio per il tuo testo.
Crea, pubblica, modifica e interagisci con le note di tutti gli utenti. YANSA è il posto giusto per condividere le tue idee e i tuoi pensieri.

Funzionalità

  • Creazione di un account
  • Accesso all'applicazione
  • Visualizzazione degli appunti
  • Visualizzazione degli account
  • Creazione di un appunto
  • Eliminazione di un appunto
  • Pubblicazione di un appunto

Un utente può creare un account e registrarsi. Dopo aver effettuato l'accesso, l'utente può seguire gli utenti, visualizzare gli appunti e creare nuovi appunti. L'utente può anche eliminare gli appunti e pubblicarli.
Si possono effettuare ricerche per rating, argomento e autore. Si può cercare direttamente il nome dell'autore.

Infrastruttura

Per la gestione del server verrà utilizzato node.js con express; utilizzeremo diversi middleware come JSONencode e URLencode per la gestione dei dati.
Ci affiancheremo ad un DB esterno, attualmente progettato in sql si potrà optare per un DB non relazionale come MongoDB.
Per la parte front-end utilizzeremo HTML, CSS e JavaScript con l'ausilio di Bootstrap per la gestione del layout e GSUP per le animazioni; e il fantastico Marocco Stefano alla grafica (non sta scrivendo lui stesso questa parte).
Per la parte di autenticazione utilizzeremo JWT oltre che l'implementazione di un sistema di crittografia per la gestione delle password e di criptazione con passaport.js.
Per la parte di testing utilizzeremo un framework tra Mocha e Jest.
Infine la parte front-end della scrittura degli appunti sarà gestita grazie ad EditorJS.

Analisi funzionale

Introduzione

Il progetto consiste nella creazione di una piattaforma web per la condivisione e la gestione di appunti tra gli utenti. La piattaforma consentirà agli utenti registrati di creare, modificare e condividere appunti, nonché interagire con gli appunti di altri utenti attraverso valutazioni anonime. Sarà inoltre possibile seguire altri utenti per visualizzare i loro appunti nel proprio profilo.


Funzionalità Principali

1. Registrazione/Login Utente Gli utenti potranno registrarsi e effettuare il login per accedere alla piattaforma.
2. Gestione degli Appunti Creazione di Appunti: Gli utenti registrati potranno creare nuovi appunti, inizialmente privati. Modifica degli Appunti: Gli utenti potranno modificare i propri appunti esistenti. Cambio di Visibilità: Gli utenti potranno modificare la visibilità dei propri appunti da privati a pubblici.
Cancellazione degli Appunti: Gli utenti potranno cancellare i propri appunti di cui sono proprietari.
Valutazione Anonima: Gli utenti potranno valutare gli appunti di altri utenti in modo anonimo.
3. Gestione dei Tag Creazione di Tag: Gli utenti potranno scegliere i tag per i propri appunti e crearne di nuovi.
4. Profilo Visualizzazione: gli utenti potranno visualizzare i profili delle altre persone. Seguire Profili: Gli utenti potranno seguire altri profili e visualizzare gli ultimi appunti pubblicati delle suddette. Verranno mostrati gli appunti pubblicati nell'ultima settimana.
5. Ricerca Ricerca per Tag: Gli utenti potranno cercare appunti utilizzando i tag. Ricerca di Profili: Gli utenti potranno cercare profili utilizzando il nome utente.


Nice to Have

Appunti Collaborativi: Possibilità di creare appunti collaborativi tra utenti.
Autenticazione con OAuth: Implementazione del login utilizzando OAuth.
Inserimento di Grafici: Possibilità di inserire grafici utilizzando Chart.js.
Importazione da File Word: Gli utenti potranno creare appunti importando file Word.
Esportazione in Word o PDF: Gli utenti potranno esportare gli appunti in formato Word o PDF.


Tecnologie e Strumenti

Editor: L'editor per la creazione e la modifica degli appunti potrebbe essere Editor.js o Quill, con preferenza per il primo.
Socket.io: Nel caso in cui si implementino gli appunti collaborativi, si potrebbe utilizzare Socket.io per la gestione in tempo reale delle modifiche.
API/Librerie per Traduzioni: Sarà necessario cercare API o librerie per tradurre i file XML (utilizzati nei file Word) in JSON, compatibile con l'editor scelto, e viceversa per l'esportazione in Word.
